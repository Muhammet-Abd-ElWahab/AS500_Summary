üìå Introduction to Probability

What is Probability?

Probability is simply a way to measure how likely something is to happen. Think of it as answering the question: "What are the chances?"

Key Points

  • Probability is measured between 0 and 1 (or 0% to 100%)
  • 0 means impossible (will never happen)
  • 1 means certain (will definitely happen)
  • 0.5 means 50-50 chance (equally likely to happen or not)

Real-Life Examples

  • Probability of the sun rising tomorrow = 1 (certain)
  • Probability of rolling a 7 on a regular die = 0 (impossible)
  • Probability of a fair coin landing on heads = 0.5 (50%)

Basic Probability Formula

For equally likely outcomes:

P(A) = Number of favorable outcomes / Total number of possible outcomes
P(A) = m/n

Example 1: Rolling a Die

What's the probability of getting an odd number?

  • Sample Space: S = {1, 2, 3, 4, 5, 6} ‚Üí n = 6
  • Odd numbers: A = {1, 3, 5} ‚Üí m = 3
  • P(A) = 3/6 = 0.5 or 50%

Example 2: Drawing from a Box

Box contains: 6 red balls and 4 white balls. What's the probability of drawing a red ball?

  • Total balls: n = 10
  • Red balls: m = 6
  • P(Red) = 6/10 = 0.6 or 60%

üî¨ Basic Probability Concepts

1. Experiment

Definition

An experiment is any action or process that produces a result.

Examples:

  • Tossing a coin
  • Rolling a die
  • Drawing a card from a deck
  • Picking a ball from a box
  • Measuring temperature

2. Sample Space (S)

Definition

The sample space is the set of all possible outcomes of an experiment.

Examples:

Experiment Sample Space Number of Outcomes
Toss a coin S = {Heads, Tails} 2
Roll a die S = {1, 2, 3, 4, 5, 6} 6
Pick a card suit S = {Hearts, Diamonds, Clubs, Spades} 4
Draw from box (6 red, 4 white) S = {Red, White} 2

3. Event (A)

Definition

An event is a specific outcome or collection of outcomes we're interested in. A subset of the sample space.

Examples:

  • Experiment: Roll a die
  • Sample Space: S = {1, 2, 3, 4, 5, 6}
  • Event A: "Getting an even number" = {2, 4, 6}
  • Event B: "Getting a number greater than 4" = {5, 6}

4. Complement of an Event (A')

Definition

All elements in the sample space that are NOT in event A.

Formula: P(A') = 1 - P(A)

Example:

  • P(getting a 6 on die) = 1/6
  • P(NOT getting a 6) = 1 - 1/6 = 5/6

5. Intersection (A ‚à© B) - "AND"

Definition

All elements that are in BOTH A and B.

Example:

Drawing a card that is both a Heart AND a King: A ‚à© B = {King of Hearts}

6. Union (A ‚à™ B) - "OR"

Definition

All elements that are in A OR B OR both.

Example:

Drawing a card that is either a Heart OR a King (includes King of Hearts)

7. Mutually Exclusive Events

Definition

Events that cannot occur together. Their intersection is empty: A ‚à© B = ‚àÖ

Example:

Getting Heads and Tails on a single coin toss are mutually exclusive.

üéØ Types of Probability

1. Classical (Objective) Probability

When to use:

When all outcomes are equally likely.

Characteristics:

  • Based on the nature of the experiment
  • Same for all observers
  • No personal opinion involved

Formula: P(A) = Number of favorable outcomes / Total number of possible outcomes

Examples:

  • Tossing a fair coin: P(Heads) = 1/2
  • Rolling a balanced die: P(getting 4) = 1/6
  • Drawing a card: P(Ace) = 4/52 = 1/13

Example in Detail: Playing Cards

Total cards in deck: 52
Number of 7s: 4 (one in each suit)
P(drawing a 7) = 4/52 = 1/13 ‚âà 0.077 or 7.7%

2. Empirical (Experimental) Probability

When to use:

Based on actual experiments or observations.

Formula: P(A) = Number of times A occurred / Total number of experiments

Example 1: Student Grades

A sample of 200 students took an exam. 10 students got "Excellent" grade.
P(Excellent) = 10/200 = 0.05 or 5%

Example 2: Birth Records

In a city, 10,000 babies were born in a year. 5,050 were boys.
P(Boy) = 5,050/10,000 = 0.505 or 50.5%

3. Subjective Probability

When to use:

Based on personal belief, experience, or judgment.

Characteristics:

  • Different for different people
  • Based on available information
  • Depends on personal circumstances

Examples:

  • "I'll attend class tomorrow with 60% probability"
    • Based on: personal schedule, weather forecast, health, etc.
  • "My team has 70% chance of winning"
    • Based on: past performance, current form, opponent strength

Difference Between Classical and Empirical Probability

Aspect Classical Probability Empirical Probability
When Calculated Before conducting experiment After conducting experiment
Based On Theory and nature of experiment Actual observed data
Method Logical reasoning Repeated trials
Example P(Heads) = 0.5 for fair coin Flip coin 1000 times, count heads

‚öôÔ∏è Basic Rules of Probability

Rule 1: Range of Probability

0 ‚â§ P(A) ‚â§ 1

Probability is always between 0 and 1 (inclusive).

Rule 2: Complement Rule

P(A) + P(A') = 1
or P(A') = 1 - P(A)

Example:

If P(Rain) = 0.3, then P(No Rain) = 1 - 0.3 = 0.7

Rule 3: Addition Rule (Union) - "OR"

P(A ‚à™ B) = P(A) + P(B) - P(A ‚à© B)

Why subtract P(A ‚à© B)?

Because when we add P(A) and P(B), we count the intersection twice. We must subtract it once to avoid double-counting.

Example: Drawing a Card

  • Event A: Drawing a Heart (13 cards) ‚Üí P(A) = 13/52
  • Event B: Drawing an Ace (4 cards) ‚Üí P(B) = 4/52
  • A ‚à© B: Ace of Hearts (1 card) ‚Üí P(A ‚à© B) = 1/52
  • P(Heart OR Ace) = 13/52 + 4/52 - 1/52 = 16/52 = 4/13

Rule 4: Addition Rule for Mutually Exclusive Events

P(A ‚à™ B) = P(A) + P(B)

When events cannot occur together, we simply add their probabilities.

Example:

Rolling a die and getting either a 2 or a 5:

  • P(2) = 1/6
  • P(5) = 1/6
  • P(2 or 5) = 1/6 + 1/6 = 2/6 = 1/3

Rule 5: Intersection (AND)

P(A ‚à© B) = P(A) √ó P(B|A)

We'll explain P(B|A) in the Conditional Probability section!

üîó Conditional Probability

Definition

Conditional probability is the probability of event A occurring given that event B has already occurred.

Notation: P(A|B) - "Probability of A given B"

Formula: P(A|B) = P(A ‚à© B) / P(B) [when P(B) > 0]

What it means:

"What's the probability of A, if we know B already happened?"

The condition changes what we consider possible!

  • Unconditional: All 52 cards possible
  • Conditional on Ace: Only 4 cards possible (the 4 Aces)

Example 1: Company Projects

A company has 100 projects:

Type IBM AT&T Total
Telecom 10 40 50
Computers 30 20 50
Total 40 60 100

Question: If we know a project is Telecommunications, what's the probability it's done by IBM?

Solution:

  • Event T: Project is Telecommunications ‚Üí P(T) = 50/100 = 0.50
  • Event I: Project is done by IBM ‚Üí P(I ‚à© T) = 10/100 = 0.10
  • P(I|T) = P(I ‚à© T) / P(T) = 0.10 / 0.50 = 0.20 or 20%

Interpretation: Of all telecom projects, 20% are done by IBM.

Example 2: Drawing Cards

Question: You draw a card and it's an Ace. What's the probability it's a Heart?

Solution:

  • Event A: Card is an Ace ‚Üí P(A) = 4/52
  • Event H: Card is a Heart ‚Üí P(H ‚à© A) = 1/52 (Ace of Hearts)
  • P(H|A) = (1/52) / (4/52) = 1/4 = 0.25 or 25%

Makes sense! Since there are 4 Aces and only 1 is a Heart, it's 1 out of 4.

üîÄ Independent vs Dependent Events - COMPLETE GUIDE

‚ö†Ô∏è THE KEY QUESTION TO ASK:

"Does the first event happening change the probability of the second event?"

  • If NO ‚Üí INDEPENDENT
  • If YES ‚Üí DEPENDENT

üü¢ INDEPENDENT EVENTS - First Event Does NOT Affect Second

Definition

Two events are INDEPENDENT if the occurrence of one does NOT affect the probability of the other.

The first event has NO EFFECT on the second event.

Mathematical Test for Independence:

Events A and B are independent if:
P(A|B) = P(A) AND P(B|A) = P(B)
OR equivalently:
P(A ‚à© B) = P(A) √ó P(B)

üìå INDEPENDENT EXAMPLES:

Example 1: Coin and Die (Independent)

  • Event A: Coin lands Heads ‚Üí P(A) = 1/2
  • Event B: Die shows 6 ‚Üí P(B) = 1/6
  • P(A ‚à© B) = P(A) √ó P(B) = 1/2 √ó 1/6 = 1/12
  • YES! Independent: The coin result doesn't affect the die, and vice versa.

Example 2: Testing Independence - Hearts and Aces

Question: Are "drawing a Heart" and "drawing an Ace" independent?

  • P(H) = 13/52 = 1/4
  • P(A) = 4/52 = 1/13
  • P(H ‚à© A) = 1/52
  • Test: Does P(H ‚à© A) = P(H) √ó P(A)?
  • 1/52 = 1/4 √ó 1/13 = 1/52 ‚úì
  • YES! They are independent.

Another way to verify:
P(H|A) = 1/4 and P(H) = 1/4 ‚Üí Since P(H|A) = P(H), they are independent!

Example 3: Tossing a Coin Twice

Independent! First toss doesn't affect second.

Example 4: Drawing Cards WITH Replacement

Independent! First card is put back, deck is unchanged.

üî¥ DEPENDENT EVENTS - First Event DOES Affect Second

Definition

Two events are DEPENDENT if the occurrence of one DOES affect the probability of the other.

Mathematical Test for Dependence:

Events A and B are dependent if:
P(A|B) ‚â† P(A) OR P(B|A) ‚â† P(B)
OR equivalently:
P(A ‚à© B) ‚â† P(A) √ó P(B)

üìå DEPENDENT EXAMPLES:

Example 1: Drawing Cards WITHOUT Replacement

Scenario: Draw 2 cards from a deck without replacing the first.

  • Event A: First card is an Ace ‚Üí P(A) = 4/52
  • Event B: Second card is an Ace
  • If A happened (first is Ace): P(B|A) = 3/51 (only 3 Aces left in 51 cards)
  • If A didn't happen (first not Ace): P(B|A') = 4/51 (still 4 Aces in 51 cards)
  • The first draw changes the second! This is dependence.

Example 2: Company Projects (Dependent)

Test: Are "Telecom" and "IBM" independent?

  • P(Telecom) = 50/100 = 0.50
  • P(IBM) = 40/100 = 0.40
  • P(Telecom ‚à© IBM) = 10/100 = 0.10
  • P(Telecom) √ó P(IBM) = 0.50 √ó 0.40 = 0.20
  • 0.20 ‚â† 0.10 ‚úó NOT independent! They are DEPENDENT.

Also: P(IBM|Telecom) = 0.20 ‚â† P(IBM) = 0.40
Knowing it's Telecom CHANGES the probability of IBM!

Example 3: Drawing Cards WITHOUT Replacement

Dependent! First draw affects what's left.

Example 4: Rain Today and Rain Tomorrow

Dependent! Weather patterns are correlated.

Example 5: Height and Shoe Size

Dependent! Taller people tend to have bigger feet.

üéØ HOW TO DECIDE: INDEPENDENT OR DEPENDENT?

STEP-BY-STEP DECISION PROCESS:

  1. Ask: "Is something removed or changed after the first event?"
    • If NO ‚Üí INDEPENDENT
    • If YES ‚Üí DEPENDENT
  2. Ask: "Does the first event change what's available for the second?"
    • If NO ‚Üí INDEPENDENT
    • If YES ‚Üí DEPENDENT
  3. Ask: "Would knowing the first result change my prediction for the second?"
    • If NO ‚Üí INDEPENDENT
    • If YES ‚Üí DEPENDENT

Scenario 1: Weather

  • Event A: It rains today
  • Event B: It rained yesterday
  • Are they independent? DEPENDENT!
  • If it rained yesterday, weather patterns suggest it might rain today

Scenario 2: Lottery Tickets

  • Event A: Your ticket wins
  • Event B: Your friend's ticket wins
  • Are they independent? INDEPENDENT!
  • Your friend winning doesn't affect your chances

Scenario 3: Job Interviews

  • Event A: You get the job
  • Event B: You pass the first interview
  • Are they dependent? DEPENDENT!
  • Passing the first interview increases your chances of getting the job

Scenario 4: Flipping Coins

  • Event A: 2nd flip is Heads
  • Event B: 1st flip is Tails
  • Are they independent? INDEPENDENT!
  • The first flip being Tails doesn't affect the second flip

üî¢ Combinatorial Concepts

Permutation

Definition

The number of ways to arrange n objects where order matters.

Formula: P(n,r) = n! / (n-r)!

Example

How many ways can 3 people be arranged in a line?

  • P(3,3) = 3! = 3 √ó 2 √ó 1 = 6 ways

Combination

Definition

The number of ways to choose n objects where order does NOT matter.

Formula: C(n,r) = n! / (r!(n-r)!)

Example

How many ways can 3 people be chosen from 5?

  • C(5,3) = 5! / (3! √ó 2!) = 120 / (6 √ó 2) = 10 ways

When to Use Each

Concept Order Matters? Use Example
Permutation YES Arrangements, sequences Race positions (1st, 2nd, 3rd)
Combination NO Selections, groups Choosing team members

‚úÖ Key Takeaways

  • Probability measures uncertainty on a scale from 0 to 1
  • Three types: Classical (objective), Empirical (experimental), and Subjective
  • Sample space contains all possible outcomes; events are subsets
  • Complement rule: P(A') = 1 - P(A)
  • Addition rule accounts for overlap: P(A ‚à™ B) = P(A) + P(B) - P(A ‚à© B)
  • Conditional probability: P(A|B) = P(A ‚à© B) / P(B)
  • Independent events: P(A ‚à© B) = P(A) √ó P(B)
  • Dependent events: P(A ‚à© B) = P(A) √ó P(B|A)
  • Permutations for arrangements (order matters); Combinations for selections (order doesn't matter)
  • Joint probability tables show probabilities for event combinations
